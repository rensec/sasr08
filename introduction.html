1 Introduction

In this tutorial, we will explore how to perform multilevel analysis on network data using R. Multilevel analysis, also known as hierarchical linear modeling, allows us to account for the interdependencies in the data.

The dependent variable in this analysis is the presence of a tie between two nodes in the network. The unit of analysis is thus the (directed) dyad, that is, a combination {from, to} of two notes. Our binary dependent variable “connected” indicates whether a tie runs from node “from” to node “to” (as measured by a name generator question in a survey, asking respondents to name their best friends).

Inderdependencies in these data come from various sources.

  1. The same node can send several ties, implying that dyads are nested within “from” nodes.

  2. The same node can receive several ties, implying that dyads are also nested within “to” nodes. Given that “from” nodes are not nested within “to” nodes and vice versa, we have a cross-classified data structure.

  3. Dyads are also nested in pairs of nodes, as the same pair of nodes can be connected in both directions (from “from” to “to” and from “to” to “from”). This implies that dyads are also nested within pairs of nodes.

  4. In these data, we have data on networks within various school classes. So, we also have nesting at the class level.

A multilevel approach to such data is the “Social Relations Model” (Snijders and Kenny, 1999). While we cannot replicate this model exactly using lme4, we can fit a model that accounts for the some sources of interdependence in the data: we account for the nesting in “senders”, “receivers”, and within pairs. What this model does not account for, in contrast to the Social Relations Model, is the fact that senders are also receivers; it treats them as separate entities, while in reality, this is the same set of nodes. If senders are also receivers, variances at the sender- and receiver level may be correlated, but unfortunately lme4 does not allow to model this correlation.
